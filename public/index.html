üí∏ Send Money (Create Payout)

POST /payouts

Send money to a bank account or mobile wallet.

üí° When to use: Use this to send money to vendors, employees, or customers. Each payout is processed automatically within 5-10 seconds.

Request Parameters:

Parameter Type Required Description
merchantReference string Required Your unique reference ID for this payout
amount number Required Amount in PKR (e.g., 1000 for Rs. 1,000)
currency string Optional Currency code (default: "PKR")
destType string Required "BANK" or "WALLET"
bankCode string Required if BANK Bank code (e.g., "HBL", "UBL")
walletCode string Required if WALLET Wallet code (e.g., "EASYPAISA")
accountNumber string Required Account number (10-16 digits)
accountTitle string Required Account holder name

Example: Send to Bank

curl -X POST https://sandbox.mycodigital.io/api/v1/payouts \
  -H "X-API-KEY: your-key" \
  -H "X-IDEMPOTENCY-KEY: unique-uuid" \
  -H "Content-Type: application/json" \
  -d '{
    "merchantReference": "ORDER-12345",
    "amount": 5000,
    "currency": "PKR",
    "destType": "BANK",
    "bankCode": "HBL",
    "accountNumber": "1234567890123",
    "accountTitle": "Ahmed Ali"
  }'

Example: Send to Mobile Wallet

curl -X POST https://sandbox.mycodigital.io/api/v1/payouts \
  -H "X-API-KEY: your-key" \
  -H "X-IDEMPOTENCY-KEY: unique-uuid" \
  -H "Content-Type: application/json" \
  -d '{
    "merchantReference": "SALARY-67890",
    "amount": 15000,
    "currency": "PKR",
    "destType": "WALLET",
    "walletCode": "EASYPAISA",
    "accountNumber": "03001234567",
    "accountTitle": "Fatima Khan"
  }'

Response:

{
  "success": true,
  "message": "Payout created successfully",
  "data": {
    "id": "payout-uuid",
    "merchantReference": "ORDER-12345",
    "amount": "5000.00",
    "status": "PENDING",
    "createdAt": "2025-11-26T10:00:00Z"
  }
}

üìä Check Payout Status

GET /payouts/:id

Check the status of a specific payout.

üí° When to use: After creating a payout, use this to check if it was successful, failed, or still pending. 
curl -H "X-API-KEY: your-key" \
  https://sandbox.mycodigital.io/api/v1/payouts/payout-uuid

Possible Status Values:

  • PENDING - Payout created, waiting to be processed
  • PROCESSING - Currently being processed
  • SUCCESS - Money sent successfully ‚úÖ
  • FAILED - Payout failed (check failureReason) ‚ùå
  • IN_REVIEW - Large amount under review
  • ON_HOLD - Account blocked or restricted

üîî Webhook Notifications

Get real-time notifications when payout status changes.

How Webhooks Work:

  1. You provide us your webhook URL when signing up
  2. When a payout status changes, we send a POST request to your URL
  3. Your server receives the notification instantly
  4. You update your system accordingly

Webhook Payload:

{
  "event": "PAYOUT_UPDATED",
  "payout": {
    "id": "payout-uuid",
    "merchantReference": "ORDER-12345",
    "amount": "5000.00",
    "status": "SUCCESS",
    "pspReference": "PSP1234567890",
    "processedAt": "2025-11-26T10:05:00Z"
  },
  "timestamp": "2025-11-26T10:05:00Z"
}

Webhook Headers:

X-MyPay-Signature: hmac-sha256-signature
X-MyPay-Event: PAYOUT_UPDATED
Content-Type: application/json
üí° Security: Always verify the X-MyPay-Signature header to ensure the webhook is from MyPay.
